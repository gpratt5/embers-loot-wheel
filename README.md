# Embers Adrift — Loot Wheels

A small self-hosted app implementing two **Suicide Kings**-style loot priority wheels for
an Embers Adrift raid team: one for **gear** drops, one for **rune** drops.

- Anyone with the link can **view** both wheels — no login needed.
- Only you (the admin) can add/remove raiders or "award" loot, which sends that raider
  to the back of the line.
- Data is stored in a SQLite file on a Render **persistent disk**, so it survives
  restarts, redeploys, and indefinitely (no expiring free database).

## How it works

Each wheel is just an ordered list of raider names. When you click **"Award → back of
line"** next to a raider, they move to the bottom of that specific wheel's list and
everyone below them moves up one spot. The gear wheel and rune wheel are completely
independent — a raider's spot on one has no effect on the other.

## 1. Push this to a GitHub repo

Create a new repo and push these files (`server.js`, `package.json`, `public/`,
`render.yaml`) to it. Render deploys straight from a Git repo.

```bash
git init
git add .
git commit -m "Initial commit: Embers Adrift loot wheels"
git branch -M main
git remote add origin https://github.com/<you>/embers-loot-wheel.git
git push -u origin main
```

## 2. Deploy on Render using the Blueprint

1. In the Render dashboard, click **New → Blueprint**.
2. Point it at your GitHub repo. Render will read `render.yaml` and provision:
   - A **Starter** web service (~$7/mo) — required because free web services on
     Render don't support persistent disks.
   - A **1 GB persistent disk** mounted at `/data` (~$0.25/GB/mo, so ~$0.25/mo extra)
     where the SQLite database lives permanently.
3. When prompted, set the **`ADMIN_PASSWORD`** environment variable to whatever
   password you want to use to edit the wheels. This is the only secret you need to set —
   `SESSION_SECRET` is auto-generated by the blueprint.
4. Click **Apply**. First deploy takes a couple of minutes.

If you'd rather click through the UI instead of using the blueprint:
1. New → Web Service → connect your repo → Environment: **Node**.
2. Plan: **Starter** (needed for the disk).
3. Add a Disk: name `loot-wheel-data`, mount path `/data`, size 1 GB.
4. Environment variables: `DATA_DIR=/data`, `ADMIN_PASSWORD=<your password>`,
   `NODE_ENV=production`.
5. Build command: `npm install`. Start command: `npm start`.

## 3. Use it

- Visit your Render URL (e.g. `https://embers-loot-wheel.onrender.com`) — that's the
  public link you share with your raid team. They'll see both wheels in read-only mode.
- Click **"Viewing only"** in the top right, enter your admin password, and you'll get
  add/remove/award/reorder controls. Your login is remembered in a session cookie for
  90 days on that browser.
- Switch between the **Gear Wheel** and **Rune Wheel** tabs — they're managed completely
  independently.

## Changing the admin password later

Render dashboard → your service → **Environment** → edit `ADMIN_PASSWORD` → save
(this triggers a redeploy automatically). Existing logged-in sessions stay valid until
they expire; if you want to force everyone out immediately after a password change,
also update `SESSION_SECRET` to a new random value at the same time.

## Local development

```bash
npm install
ADMIN_PASSWORD=test123 npm start
```

Then open `http://localhost:3000`. Data is stored in `./data/wheels.db` locally.

## Backups (optional but recommended)

Even with a persistent disk, it's good practice to back up occasionally. You can
download the live database via Render's Shell tab:

```bash
# In the Render Shell for your service
cat /data/wheels.db | base64
```
Copy the output and decode it locally, or use Render's `scp`-style shell file access
if enabled on your plan.
